This answer demonstrates the full and apropiate process of adding a object to an ASP.NET Project

  1. Added controller. (NOTE: I am using Visual Studios 2022 Comunity Edition.)
  2. Used the empty controller template and named it PersonController.
  3. Defined the Person class and the Contact class and added them to the Models folder provided by default in a ASP.NET MVC Web Application.

  4. Declared and instantiated the static List Persons class.
  5. Added a JSON file to act as the persistance layer (database), and place it in a folder named data in the wwwroot section of the solution. Made sure JSON was valid.


  6. Add Services folder to the solution, define a class entitled JsonPersonService and added it to the Services folder. The JsonPersonService deserializes the JSON file for consumption.
  7. Registered the Service in the Program.cs file. NOTE: The Startup.cs no longer exists in an ASP.NET solution. All configurations are done in the Program file in .NET 6.
  8. Passed the Service into the Controller, invoked the Action to populate a list and pass the list on the View
  9. Entered the following path (localhost:port).../Person/PersonList in the browser address bar. Ran the application, and success!.
  10. Now, I get to the core answer, which is to add a new Person object to the list. However, I went beyond and added the Person object to a persistance layer (persons.json).
  11. I defined the Action provided (Pretty, much exactly as the bare answer above).
  12. Add the Razor sytax provided. But where? I decided for the Index View.
  13. Navigate to the Person/Index, populated the User and the Contact field, and posted it.
  14. Success!, like above, the Person object is added to the Persons list.
